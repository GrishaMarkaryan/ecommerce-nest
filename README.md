## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Installation

```bash
$ pnpm install
```

## Running the app

```bash
# development
$ pnpm run start

# watch mode
$ pnpm run start:dev

# production mode
$ pnpm run start:prod
```

## Test

```bash
# unit tests
$ pnpm run test

# e2e tests
$ pnpm run test:e2e

# test coverage
$ pnpm run test:cov
```

# Project Setup Guide

Welcome to our Express.js project. This guide is designed to help you set up the project locally, including steps for database installations and environment configuration.

## Prerequisites

Before you begin, ensure you have the following installed:
- [Node.js](https://nodejs.org/en/) (LTS version recommended)
- [npm](https://www.npmjs.com/) or [Yarn](https://yarnpkg.com/) or [pnpm](https://pnpm.io/)
- Database management tools for the two databases used in this project

## Database Setup

This project relies on two databases. Follow the steps below to install and launch them locally:

### MongoDB Shell

1. Download and install MongoDB Shell from its official [website](https://www.mongodb.com/try/download/shell).
2. Launch and create a new database instance named `e-commerce`.
```
mongosh
use e-commerce
```
3. Import the initial data.
```
ts-node scripts/resetData.ts
```
#### Useful commands
```
show collections
db.users.find()
```

### Redis

1. Download and install Redis from [GitHub](https://github.com/tporadowski/redis/releases).
2. Launch Redis
```
redis-server &
```

#### Useful commands
```
redis-cli
KEYS *
flushall
```


## Environment Configuration

You need to set up an `.env` file at the root of the project to store environment variables:

1. Copy the `.env.development` file to a new file named `.env`.
2. Replace the asterisks and fill in the database connection details and any other environment-specific variables as documented in the `.env.development` file.
